# Sub2API Xray v0.1.157-xray1 リリースノート

リリース日：2026-07-16

上流ベース：[Sub2API v0.1.157](https://github.com/Wei-Shaw/sub2api/releases/tag/v0.1.157)

Sub2API Xray 独自のバージョンおよび更新チャネルとして最初のリリースです。上流 `v0.1.157` に、ユーザー専用リソース管理、サブスクリプション配布、組み込み Xray プロキシランタイムを追加しています。

## ハイライト

- 一般ユーザーが管理者 API を利用せずに使える、完全な専用リソース画面を追加しました。
- 標準プロキシと Xray プロキシプール、ノードのインポート、プロキシプロバイダーのサブスクリプション同期を追加しました。
- ユーザー自身によるサブスクリプション割り当てと交換コード配布を追加しました。
- アカウントプールのヘルス表示、購読解除、アカウント利用履歴、上流エラー詳細を追加しました。
- 新しいユーザー画面を管理者 UI に揃え、モーダル、テーブル、小さい画面での表示を改善しました。

## 追加機能

### ユーザー専用リソース

- マイグループ：ルーティング、クォータ、倍率、RPM、モデル、fallback、プラットフォーム設定を含む専用グループを管理できます。
- マイアカウント：プラットフォーム別の接続、テスト、更新、インポート/エクスポート、一括操作を備えた専用アカウントプールを管理できます。
- マイプロキシ：専用プロキシを管理し、認証情報を受け取ることなく管理者公開プロキシを選択できます。
- サブスクリプション割り当て：専用グループを使って割り当て、延長、リセット、取り消し、復元ができます。
- マイ交換コード：複数回利用コードと交換詳細を含むサブスクリプション交換コードを配布できます。

### Xray プロキシランタイム

- `vmess`、`vless`、`trojan`、Shadowsocks、SOCKS、HTTP ノードリンクに対応します。
- Base64 ノードリストと一般的な Clash YAML サブスクリプションに対応します。
- Xray ノードごとに管理されたローカル SOCKS インバウンドを作成し、アカウント通信とプロキシテストで同じ経路を使用します。
- 一括インポートでは有効なノードを個別に保存し、1件のエラーで全体を破棄せず行単位のエラーを返します。

### サブスクリプションと診断

- 利用可能、レート制限中、合計のアカウントプール数を表示します。
- 購読者が自分のサブスクリプションを解除できます。
- 自分の利用履歴と、自分が所有するアカウントの利用履歴を分けて確認できます。
- 現在のユーザーのリソースに限定し、機密情報を秘匿化した上流エラー詳細を表示します。

## セキュリティとプライバシー

- 非公開グループ、アカウント、プロキシ、交換コード、プロキシソース、管理対象サブスクリプションは、サービス層とリポジトリ層の両方で所有者ごとに分離されます。
- 一般ユーザーはアカウント、ルート、fallback グループを他ユーザーの非公開リソースへ関連付けできません。
- 公開プロキシのパスワード、認証情報、ノードシークレットは一般ユーザーへ返しません。
- 割り当て、取り消し、復元、交換、購読解除の後にサブスクリプションと請求キャッシュを無効化します。
- ユーザーを無効化すると専用リソースと配布経路も停止し、システムリソースには変換しません。
- ユーザーリソース画面は `enable_user_resources` で制御され、アップグレード環境ではデフォルトで無効です。

## ブランドと更新

- 製品名を `Sub2API Xray` に変更しました。
- バージョン形式を `v<upstream>-xray<revision>` に変更しました。
- 更新確認、ロールバック手順、リリースリンク、コンテナイメージは `SMNNagarajan/sub2api-xray` を使用します。
- コンテナイメージは `ghcr.io/smnnagarajan/sub2api-xray` です。

## アップグレード手順

1. PostgreSQL、Redis、設定、プロキシサブスクリプションのシークレットをバックアップします。
2. 通常のデータベースマイグレーションを実行します。既存リソースはシステム所有のままで、自動的にユーザーへ割り当てられません。
3. Xray ランタイムを設定するか、xray-core を含む本プロジェクトの Docker イメージを使用します。
4. `enable_user_resources` を有効にする前に、管理者アクセスと既存リソースの所有権を確認します。
5. 有効化後は一般ユーザーで専用プロキシまたはグループを作成し、呼び出しテストを行ってから利用範囲を広げます。

Xray 専用マイグレーションとユーザー所有データを確認せずに、上流機能のみのビルドへダウングレードしないでください。
