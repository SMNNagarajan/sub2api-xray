# Sub2API Xray v0.1.161-xray2 Fixes and Updates

Release date: 2026-07-22

## Upstream synchronization

- Merged the upstream v0.1.161 line while retaining its gateway, account, image, operations, authentication-cache, and ingress-security updates.
- Xray-specific database changes use migration `185`, after upstream migrations `183` and `184`.

## Proxy subscriptions and protocols

- Imports accept plain share links, Base64 subscriptions, Clash/Mihomo YAML, and sing-box JSON containing `outbounds` or `endpoints`.
- Standard proxies support HTTP, HTTPS, SOCKS, SOCKS5, and SOCKS5H.
- Xray-class proxies support VMess, VLESS, Trojan, and Shadowsocks. Xray Core handles the first three with common TCP, WebSocket, gRPC, HTTPUpgrade, XHTTP, TLS, and REALITY combinations.
- A controlled sing-box runtime handles Shadowsocks and adds Hysteria, Hysteria2/Hy2, TUIC, AnyTLS, Naive (HTTPS/QUIC), and WireGuard/WG support.
- Common aliases and share-link formats are normalized. A malformed node reports its own error without preventing valid nodes from importing.
- The Docker image includes both Xray Core and sing-box and exposes them through local SOCKS inbounds for account traffic, connection tests, and quality checks.

## My Proxies

- Added standard, batch, share-link, full-config, and provider-subscription creation modes aligned with the administrator IP Management experience.
- Proxy sources can be created, edited, synchronized manually, and assigned refresh intervals; synchronization creates, updates, and retires nodes.
- Users can publish their own proxies or make nodes synchronized from a proxy source public. Other users cannot see the subscription URL and may only select and use the resulting public nodes.
- Public proxies hide host, port, username, password, raw node data, fallback details, and other sensitive fields from non-owners.
- Owned or public proxies may be used as child/fallback proxies, with direct fallback also available. Database rules reject private cross-owner references and self-reference.
- Added individual and batch connection tests, individual and batch quality checks, progress reporting, and persisted quality results.
- Xray and sing-box checks run in isolated temporary runtime instances that are cleaned up after each probe.
- Imported and synchronized nodes automatically receive one queued quality check through a bounded worker pool.
- Search, filters, column settings, location/quality/account-count fields, action menus, and wide-table layout now match IP Management more closely.

## My Accounts and administrator account management

- My Accounts now reuses the administrator create and edit dialogs with user-scoped groups, proxies, platforms, and account types.
- Completed the upstream-style OAuth, API Key, Setup Token, Cookie, Service Account, and Codex Session/PAT entry and import flows.
- Added token refresh, error clearing, usage navigation, scheduling toggles, bulk refresh, bulk clear, bulk edit, import, and export actions.
- Filters, column settings, capacity, status, usage windows, proxy, priority, multiplier, and timestamp columns now align with Account Management.
- Fixed administrator proxy changes: system accounts may use only system proxies, while user accounts may use only owned or public proxies.
- Connection testing now reuses the full administrator modal with model selection, OpenAI modes, live SSE output, retries, and image previews.
- The test prompt is editable and defaults to `hi`. It works across OpenAI, Claude, Gemini, Grok, Bedrock, Vertex, and Antigravity; Compact mode hides the prompt.
- Fixed missing account-test and token-refresh service injection that made the user test endpoint unavailable.

## Tables and responsive dialogs

- Added a persistent draggable horizontal scrollbar to the shared wide-table component with bidirectional content synchronization.
- A fallback drag bar appears when the native scrollbar is hidden, with Arrow, Page Up/Down, Home, and End keyboard controls.
- Restored horizontal access and action-column visibility across Account, IP, User, and Group Management plus My Accounts and My Proxies.
- Improved dialog height, internal scrolling, long-account-name truncation, and mobile layouts so controls and form content are not clipped.

## Additional fixes and security hardening

### User-resource lifecycle and transaction consistency

- Deleting or disabling a user now deactivates owned groups, accounts, proxies, proxy sources, and redeem codes and revokes managed subscriptions in one database transaction. Content-moderation auto-bans use the same lifecycle path.
- Subscriber-initiated unsubscribe operations record the revoking user. Resource owners cannot reactivate a subscription that the subscriber explicitly left; administrators retain an explicit force-restore path.
- Subscription authorization changes enqueue API-key cache invalidations in the database outbox so Redis or cross-instance notification failures cannot preserve stale access.
- Group rate/RPM overrides, account fields and memberships, proxy-source synchronization, stale-node disabling, and scheduler notifications now commit atomically and roll back together on failure.
- User-resource delete, clear-error, refresh, bulk mutation, and proxy-source synchronization operations enqueue scheduler changes inside their database transactions.

### Ownership, privacy, and network security

- Administrator account create, edit, and bulk-edit flows validate account fields, group ownership, and proxy ownership before committing all related changes atomically.
- Public proxies cannot be made private or deleted while referenced by another account or fallback proxy. Fallback references are included in dependency counts and cache invalidation.
- Public-proxy responses redact credentials and node material across modern sharing formats, while proxy client cache keys retain only a digest of authentication data.
- User-owned accounts enforce public-network-only targets during validation, model preview, and real requests, blocking loopback, private, link-local, and cloud metadata destinations. System accounts retain upstream-compatible custom URL behavior.
- Deployment URL-allowlist defaults now match upstream, so custom public API hosts are no longer rejected by a static host list while user-owned SSRF controls remain active.
- Credential-bearing user-resource routes omit request bodies from audit logs, and API-key authentication rejects private standard groups owned by another user.

### Account capabilities and gateway compatibility

- My Accounts adds upstream-model synchronization, pre-create model preview, Gemini OAuth capability discovery, and Antigravity default model mappings under owner-scoped APIs.
- Account create/edit dialogs load models, proxies, and OAuth capabilities by administrator or user scope and improve Codex import results, Gemini OAuth options, and model-whitelist interactions.
- Administrator Account Management and My Accounts now use the same connection-test dialog. Text prompts are editable with a `hi` default, while Compact mode still hides the prompt.
- Added Grok `/messages/count_tokens` compatibility, Anthropic request token estimation, rolling 24-hour free-tier quota recognition, and retry handling for billing probes.
- Improved Grok multi-turn cache identity, Claude Code session detection, one-time fallback for invalid encrypted reasoning, and bidirectional Anthropic thinking-signature/Responses reasoning conversion.
- Content-moderation bans, user state, and private-resource deprovisioning remain atomic; a lifecycle transaction failure no longer leaves active resources behind a partially changed user status.

### Build and deployment

- Added migration `186_user_resource_lifecycle_hardening.sql` for revocation attribution, fallback-proxy constraints, and subscription authorization-cache outbox invalidation.
- Docker build contexts now ignore the local `.pnpm-store`, preventing dependency caches from being uploaded into builds.

## Upgrade notes

1. Back up PostgreSQL, Redis, configuration, and proxy subscription secrets.
2. This update includes `185_user_public_proxies.sql` and `186_user_resource_lifecycle_hardening.sql`, adding proxy-source visibility, account/fallback ownership constraints, revocation attribution, and authorization-cache outbox invalidation.
3. The Docker image includes Xray Core `26.3.27` and sing-box `1.13.14`. Standalone deployments must configure `XRAY_BIN` and `SING_BOX_BIN`.
4. After deployment, verify administrator proxy changes, ordinary-user proxy imports, public proxy selection, and account connection tests.
5. The user resource workspace remains controlled by `enable_user_resources` and defaults to disabled on upgraded installations.
6. Do not downgrade to an upstream-only build before reviewing the migration and user-owned resource data.
