# Sub2API Xray v0.1.161-xray2

Release date: 2026-07-20

Upstream baseline: [Sub2API v0.1.161](https://github.com/Wei-Shaw/sub2api/releases/tag/v0.1.161) (`b8e844f4e`)

This release updates the Xray branch to the latest upstream `main` while retaining the user-owned resource workspace and proxy runtime. The Xray revision remains `2` because this update contains no new Xray-only release series.

## Changes

- Merged the upstream v0.1.161 line, including the current gateway, account, image, operations, authentication-cache, and ingress-rejection changes.
- Preserved private user groups, accounts, proxies, proxy sources, assigned subscriptions, and redeem-code ownership isolation.
- Preserved standard and Xray-backed proxy support, protocol imports, public-proxy credential redaction, connection tests, and quality checks.
- Added the next migration slot for user-public-proxy constraints after upstream migrations `183` and `184`.
- Kept the `Sub2API Xray` brand and `v<upstream>-xray<revision>` version format separate from upstream update checks.

## Compatibility and security

- Existing resources remain system-owned unless explicitly assigned; no historical resource is reassigned during upgrade.
- Ordinary users continue to use `/my/*` resource APIs and cannot use administrator resource endpoints to access another user's data.
- Public proxy responses do not expose passwords, node secrets, or other private authentication fields.
- Subscription, billing, and API-key authentication caches retain the invalidation behavior from both branches.
- The user resource workspace remains controlled by `enable_user_resources`, which defaults to disabled for upgraded installations.

## Upgrade notes

1. Back up PostgreSQL, Redis, configuration, and proxy subscription secrets.
2. Deploy the image or binary built from this release and allow the embedded migrations to run.
3. Confirm administrator access and resource ownership before enabling user resources.
4. Test one administrator account and one ordinary user account before opening the service to normal traffic.

Do not downgrade to an upstream-only build without reviewing the Xray-specific migrations and user-owned resource data.

## Validation

- Frontend: `vue-tsc --noEmit`, `pnpm test:run` (`188` files, `1272` tests), and `pnpm build` passed.
- The final Docker image and health endpoints were validated on the review server before deployment.
