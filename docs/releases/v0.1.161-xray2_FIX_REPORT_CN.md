# Sub2API Xray v0.1.161-xray2 累计修复报告

本报告汇总 `v0.1.161-xray2` 在正式发布前完成的用户资源、代理运行时、账号管理、网关兼容和安全加固。Release 正文保留面向使用者的变化，本文件记录问题指向、修复方式和部署影响。

## 1. 自定义 API 渠道被错误拦截

### 问题

- 管理员账号和用户账号使用自定义公网 API 地址时，连接测试返回 `host is not allowed`。
- 账号凭据、模型和上游服务尚未被请求，错误发生在本地 URL 校验阶段。

### 原因

- review Compose 配置错误地强制开启全局静态 URL 白名单，与官方 Docker 默认值不一致。
- 自定义公网域名不在静态主机表中，因此所有渠道在发出请求前被拒绝。

### 修复

- 部署配置恢复官方 URL 白名单默认值，允许管理员配置自定义公网上游。
- 用户私有账号继续通过独立网络策略拒绝环回、私网、链路本地和云元数据地址。
- 网络策略同时在 URL 验证和实际拨号阶段执行，避免 DNS 变化绕过检查。

## 2. 管理员连接测试提示词不可编辑

### 问题

- “我的账号”连接测试可以编辑提示词，管理员“账号管理”仍显示旧弹窗，只在图片模型下显示输入框。
- 普通文本测试固定提交空提示词，与用户侧体验和后端能力不一致。

### 原因

- 前端同时保留了管理员专用旧组件和支持作用域的新共享组件，管理员页面仍引用旧实现。

### 修复

- 管理员组件改为复用共享 `AccountTestModal`，并固定传入 `scope=admin`。
- 管理员请求仍调用 `/admin/accounts/*` 并携带管理员 UI 请求头；用户请求仍调用 `/my/accounts/*`。
- 普通测试提示词默认 `hi` 且可编辑，Compact 模式隐藏提示词，图片模型继续提供图片提示词和预览。

## 3. 用户资源生命周期不完整

### 问题

- 删除、停用或自动封禁用户时，用户状态与其私有资源可能分步更新。
- 任一步骤失败可能留下仍可调度的账号、代理、兑换码或订阅。
- 资源所有者可能恢复订阅者已经主动退订的订阅。

### 修复

- 管理员删除/停用和内容审核自动封禁统一调用原子生命周期流程。
- 同一事务中停用用户私有分组、账号、代理、代理订阅源、兑换码，并撤销其管理的订阅。
- `user_subscriptions` 增加 `revoked_by_user_id`，区分订阅者主动退订与管理者撤销。
- 管理者不能恢复订阅者主动退出的订阅；管理员保留显式强制恢复能力。
- 生命周期数据库不可用或事务失败时，拒绝执行部分更新。

## 4. 订阅鉴权缓存可能短暂过期

### 问题

- 订阅状态、分组、有效期或删除状态变化不会修改 API Key 行。
- Redis 或实例间通知异常时，其他实例可能继续使用旧订阅授权缓存。

### 修复

- 迁移 `186_user_resource_lifecycle_hardening.sql` 为订阅变更增加数据库触发器。
- 触发器按用户查找有效 API Key，将哈希后的缓存键写入 `auth_cache_invalidation_outbox`。
- 数据库提交与缓存失效事件保持同一事务语义，后台可重试投递。

## 5. 账号、分组与代理所有权缺少原子边界

### 问题

- 管理员创建、编辑或批量编辑用户账号时，账号字段、分组关系和代理可能分开写入。
- 错误的跨用户分组或私有代理引用可能在部分数据写入后才被发现。

### 修复

- 写入前统一校验账号 owner、分组 owner、代理 owner 和公开状态。
- 账号字段、分组关系、代理变更和调度通知在同一事务内保存。
- 创建或更新任一步失败时回滚全部修改。
- API Key 鉴权拒绝访问不属于当前用户的私有标准分组。

## 6. 公开代理和备用代理依赖保护不足

### 问题

- 公开代理除直接绑定账号外，还可能被其他代理作为备用代理引用。
- 取消公开、删除代理或质量状态变化时，备用引用没有全部纳入依赖和缓存刷新。

### 修复

- 数据库约束同时检查账号 `proxy_id` 和代理 `backup_proxy_id`。
- 被其他用户资源引用的公开代理禁止取消公开或删除。
- 备用代理用户纳入依赖计数、探测结果清理和调度缓存失效。
- 公开代理响应清理现代分享协议中的节点密钥、认证字段和嵌套配置。
- HTTP 客户端缓存键仅保存代理认证信息摘要，不保存可还原凭据。

## 7. 代理订阅同步的事务与运行时清理

### 问题

- 订阅同步包含解析、创建、更新、停用旧节点、清理探测结果和调度通知。
- 中途失败可能留下半同步节点或未清理的临时 Xray/sing-box 运行实例。

### 修复

- 节点创建、更新、缺失节点停用、同步状态和调度 outbox 改为事务写入。
- 同步失败时记录安全摘要，不回显订阅凭据和节点密钥。
- Xray 与 sing-box 的临时探测实例设置数量上限，并在成功、失败和取消路径统一清理。
- 删除订阅源、质量检测和批量导入均补充回滚与清理覆盖。

## 8. 用户账号模型能力与原版体验不一致

### 修复

- 新增用户作用域的上游模型同步、创建前模型预览、Gemini OAuth 能力和 Antigravity 默认模型映射接口。
- 创建/编辑账号组件按 `admin` 或 `user` 作用域加载模型、代理、分组和 OAuth 能力。
- 用户模型同步接口强制校验账号所有权，不接受其他用户账号 ID。
- 创建前预览使用临时账号对象并应用用户网络策略，不持久化 API Key。
- 完善 Codex Session/PAT 导入结果、Gemini OAuth 参数和模型白名单编辑体验。

## 9. Grok 与 Anthropic/Responses 兼容补充

### 修复

- 增加 Grok `/messages/count_tokens` 路由和 Anthropic 请求 token 估算。
- 识别 Grok 免费账号滚动 24 小时 `1,000,000` token 限额。
- 计费探测对可恢复状态进行有限重试，并限制错误正文长度。
- Grok 多轮缓存优先使用 Claude Code session 标识，避免用户文本变化造成缓存身份漂移。
- 上游无法解密 `encrypted_content` 时，最多执行一次移除加密 reasoning 的降级重试。
- Anthropic thinking signature 与 OpenAI Responses reasoning 加密内容支持双向转换。

## 10. 构建与部署可靠性

### 修复

- `.dockerignore` 排除根目录 `.pnpm-store`，避免本地依赖缓存进入 Docker 构建上下文。
- 正式版本号保持 `v0.1.161-xray2`；本地备份使用 `-betaN` 标记，正式发布自动去除 beta 后缀。
- 数据库新增迁移 `186`，升级顺序保持在 Xray 专属迁移 `185` 之后。

## 数据库影响

- 新增 `user_subscriptions.revoked_by_user_id`，外键删除策略为 `SET NULL`。
- 新增订阅撤销人索引。
- 更新公开代理撤销/删除触发器，覆盖备用代理引用。
- 新增订阅鉴权缓存失效触发器和 outbox 写入。
- 历史软删除订阅会保守回填为订阅者主动撤销，避免被资源所有者静默恢复。

## 升级注意事项

1. 升级前备份 PostgreSQL、Redis、部署配置和代理订阅密钥。
2. 确认迁移 `185`、`186` 均执行成功。
3. 部署后核对用户停用、订阅退订、公开代理取消公开、账号代理切换和自定义公网 API 连接测试。
4. 不要在未处理用户私有资源数据的情况下回退到不包含 Xray 用户资源迁移的构建。
