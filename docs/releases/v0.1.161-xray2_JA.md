# Sub2API Xray v0.1.161-xray2 リリースノート

リリース日：2026-07-20

上流ベース：[Sub2API v0.1.161](https://github.com/Wei-Shaw/sub2api/releases/tag/v0.1.161)（`b8e844f4e`）

本リリースでは Xray ブランチを上流の最新 `main` に更新し、ユーザー専用リソース画面とプロキシランタイムを維持しました。今回 Xray 固有の新しいリリース系列は追加していないため、Xray リビジョンは `2` のままです。

## 変更内容

- ゲートウェイ、アカウント、画像、運用、認証キャッシュ、入口拒否処理を含む上流 v0.1.161 系列を統合しました。
- ユーザー専用グループ、アカウント、プロキシ、プロキシソース、割り当てサブスクリプション、交換コードの所有者分離を維持しました。
- 標準および Xray プロキシ、プロトコルインポート、公開プロキシの認証情報秘匿、接続テスト、品質検査を維持しました。
- 上流の `183`、`184` マイグレーションの後ろに新しい番号を割り当て、ユーザー公開プロキシの制約を維持しました。
- `Sub2API Xray` ブランドと `v<upstream>-xray<revision>` 形式を維持し、上流の更新チャネルと分離しました。

## 互換性とセキュリティ

- 既存リソースはシステム所有のままで、アップグレード時に自動でユーザーへ割り当てられません。
- 一般ユーザーは引き続き `/my/*` リソース API を使用し、他ユーザーのデータを管理者 API 経由で取得できません。
- 公開プロキシのレスポンスにはパスワード、ノードシークレット、その他の非公開認証情報を含めません。
- サブスクリプション、請求、API Key 認証キャッシュの無効化処理を両ブランチの統合後も維持しました。
- ユーザーリソース画面は `enable_user_resources` で制御され、アップグレード環境ではデフォルトで無効です。

## アップグレード手順

1. PostgreSQL、Redis、設定、プロキシサブスクリプションのシークレットをバックアップします。
2. 本リリースのイメージまたはバイナリをデプロイし、組み込みマイグレーションの完了を待ちます。
3. ユーザーリソースを有効にする前に、管理者アクセスと所有権を確認します。
4. 管理者アカウントと一般ユーザーアカウントで基本操作を確認してから通常トラフィックを戻します。

Xray 固有のマイグレーションとユーザー所有データを確認せずに、上流のみのビルドへダウングレードしないでください。
