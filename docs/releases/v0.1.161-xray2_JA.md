# Sub2API Xray v0.1.161-xray2 修正および更新

リリース日：2026-07-22

## 上流との同期

- 上流 v0.1.161 を統合し、ゲートウェイ、アカウント、画像、運用、認証キャッシュ、入口セキュリティの更新を保持しました。
- Xray 固有のデータベース変更には、上流の `183`、`184` に続くマイグレーション `185` を使用します。

## プロキシ購読とプロトコル

- 通常の共有リンク、Base64 購読、Clash/Mihomo YAML、`outbounds` または `endpoints` を含む sing-box JSON をインポートできます。
- 標準プロキシは HTTP、HTTPS、SOCKS、SOCKS5、SOCKS5H に対応します。
- Xray 系プロキシは VMess、VLESS、Trojan、Shadowsocks に対応します。Xray Core は最初の 3 種類を処理し、一般的な TCP、WebSocket、gRPC、HTTPUpgrade、XHTTP、TLS、REALITY の組み合わせに対応します。
- 管理された sing-box ランタイムが Shadowsocks を処理し、Hysteria、Hysteria2/Hy2、TUIC、AnyTLS、Naive（HTTPS/QUIC）、WireGuard/WG にも対応します。
- 一般的な別名と共有リンク形式を正規化します。不正なノードは個別にエラーを返し、他の有効なノードの取り込みを妨げません。
- Docker イメージには Xray Core と sing-box の両方を同梱し、アカウント通信、接続テスト、品質検査ではローカル SOCKS インバウンドを使用します。

## マイプロキシ

- 標準、バッチ、共有リンク、完全設定、プロバイダー購読の作成モードを追加し、管理者の IP 管理に操作感を合わせました。
- プロキシ購読元の作成、編集、手動同期、更新間隔設定に対応し、同期時にノードを追加、更新、停止します。
- 自分のプロキシ、または購読元から同期されたノードを全ユーザー向けに公開できます。他のユーザーには購読 URL を表示せず、公開ノードの選択と利用だけを許可します。
- 非所有者には公開プロキシのホスト、ポート、ユーザー名、パスワード、元ノード、フォールバック情報などを表示しません。
- 所有プロキシまたは公開プロキシを子/予備プロキシとして利用でき、直接接続へのフォールバックにも対応します。非公開の所有者越境参照と自己参照はデータベースで拒否します。
- 単体/一括の接続テストと品質検査、進捗表示、品質結果の保存を追加しました。
- Xray と sing-box の検査は分離された一時ランタイムで実行し、完了後に自動削除します。
- インポートまたは同期したノードは、有界キューと固定ワーカーによって品質検査を一度自動実行します。
- 検索、フィルター、列設定、位置/品質/アカウント数、操作メニュー、ワイドテーブルを IP 管理に合わせて改善しました。

## マイアカウントと管理者アカウント管理

- マイアカウントで管理者用の作成/編集ダイアログを再利用し、グループ、プロキシ、プラットフォーム、アカウント種別をユーザー範囲に限定しました。
- OAuth、API Key、Setup Token、Cookie、Service Account、Codex Session/PAT の入力とインポートを上流 UI に合わせました。
- トークン更新、エラー解除、利用履歴、スケジュール切替、一括更新、一括エラー解除、一括編集、インポート、エクスポートを追加しました。
- フィルター、列設定、容量、状態、使用量ウィンドウ、プロキシ、優先度、倍率、日時列をアカウント管理に合わせました。
- 管理者によるプロキシ変更を修正し、システムアカウントはシステムプロキシのみ、ユーザーアカウントは所有または公開プロキシのみ選択可能にしました。
- 接続テストは管理者と同じ完全版ダイアログを使用し、モデル選択、OpenAI モード、SSE 出力、再試行、画像プレビューに対応します。
- テストプロンプトは編集可能で、既定値は `hi` です。OpenAI、Claude、Gemini、Grok、Bedrock、Vertex、Antigravity に対応し、Compact モードでは非表示になります。
- アカウントテストサービスとトークン更新サービスの注入不足によりユーザー用テスト API が利用できなかった問題を修正しました。

## テーブルとレスポンシブ表示

- 共通ワイドテーブルに常時表示されるドラッグ可能な横スクロールバーを追加し、内容と双方向に同期します。
- ネイティブスクロールバーが非表示の場合は代替バーを表示し、矢印、Page Up/Down、Home、End キーにも対応します。
- アカウント、IP、ユーザー、グループ管理と、マイアカウント、マイプロキシで横方向の閲覧と操作列表示を修復しました。
- ダイアログ高さ、内部スクロール、長いアカウント名の省略、モバイル配置を改善し、操作やフォームが切れないようにしました。

## 追加修正とセキュリティ強化

### ユーザーリソースのライフサイクルとトランザクション整合性

- ユーザーの削除または無効化時に、所有グループ、アカウント、プロキシ、購読元、交換コードの停止と管理中購読の取消を 1 つのデータベーストランザクションで実行します。コンテンツ審査による自動停止も同じ処理を使用します。
- 購読者自身の解除では取消実行者を記録し、リソース所有者が明示的に退出した購読を再開できないようにしました。管理者は明示的な強制復元を引き続き利用できます。
- 購読権限の変更は API Key 認証キャッシュ無効化をデータベース outbox に登録し、Redis やインスタンス間通知の障害時にも古い権限を残しません。
- グループ倍率/RPM 上書き、アカウント項目と所属、購読元同期、欠落ノード停止、スケジューラ通知を原子的に保存し、途中失敗時は全体をロールバックします。
- ユーザーリソースの削除、エラー解除、更新、一括変更、購読元同期は同じトランザクション内でスケジューラ outbox を更新します。

### 所有権、プライバシー、ネットワーク保護

- 管理者のアカウント作成、編集、一括編集では、項目、グループ所有権、プロキシ所有権を事前検証し、関連変更を同一トランザクションで保存します。
- 他のアカウントまたは予備プロキシから参照中の公開プロキシは非公開化または削除できません。予備参照も依存数とキャッシュ無効化に含めます。
- 公開プロキシ応答では最新の共有形式を含めて認証情報とノード情報を除去し、プロキシクライアントのキャッシュキーには認証情報のダイジェストだけを保持します。
- ユーザー所有アカウントは、検証、モデルプレビュー、実リクエストの各段階で公開ネットワークのみを許可し、ループバック、プライベート、リンクローカル、クラウドメタデータ宛先を拒否します。システムアカウントは上流互換のカスタム URL 動作を保持します。
- 配置時の URL 許可リスト既定値を上流に合わせ、公開カスタム API ホストを静的リストで誤拒否しないようにしました。ユーザー所有リソースの SSRF 保護は引き続き有効です。
- 認証情報を含むユーザーリソース API は監査ログに本文を保存せず、API Key 認証は他ユーザー所有の非公開標準グループを拒否します。

### アカウント機能とゲートウェイ互換性

- マイアカウントに上流モデル同期、作成前モデルプレビュー、Gemini OAuth 機能取得、Antigravity 既定モデルマッピングを所有者範囲 API として追加しました。
- 作成/編集ダイアログは管理者またはユーザー範囲でモデル、プロキシ、OAuth 機能を読み込み、Codex インポート結果、Gemini OAuth、モデル許可リスト操作を改善しました。
- 管理者アカウント管理とマイアカウントは同じ接続テストダイアログを使用します。通常テストのプロンプトは `hi` を既定値として編集でき、Compact モードでは非表示になります。
- Grok `/messages/count_tokens` 互換、Anthropic リクエストの token 推定、無料枠のローリング 24 時間判定、課金プローブの再試行を追加しました。
- Grok の複数ターンキャッシュ識別、Claude Code session 検出、暗号化 reasoning エラー時の 1 回限りの縮退再試行、Anthropic thinking signature と Responses reasoning の双方向変換を改善しました。
- コンテンツ審査停止、ユーザー状態、所有リソース停止を原子的に保ち、ライフサイクルトランザクション失敗時に一部だけ状態が変わらないようにしました。

### ビルドと配置

- `186_user_resource_lifecycle_hardening.sql` を追加し、購読取消の実行者、予備プロキシ制約、購読認証キャッシュ outbox 無効化を補強しました。
- Docker ビルドコンテキストからローカル `.pnpm-store` を除外し、依存キャッシュの不要なアップロードを防止しました。

## アップグレード手順

1. PostgreSQL、Redis、設定ファイル、プロキシ購読シークレットをバックアップします。
2. `185_user_public_proxies.sql` と `186_user_resource_lifecycle_hardening.sql` が含まれ、購読元の公開フラグ、アカウント/フォールバック所有者制約、取消実行者、認証キャッシュ outbox 無効化が追加されます。
3. Docker イメージには Xray Core `26.3.27` と sing-box `1.13.14` が含まれます。単体バイナリ構成では `XRAY_BIN` と `SING_BOX_BIN` を設定してください。
4. 配置後、管理者のプロキシ変更、一般ユーザーのプロキシ取り込み、公開プロキシ選択、アカウント接続テストを確認してください。
5. ユーザーリソース画面は `enable_user_resources` で制御され、既存環境のアップグレード時は既定で無効です。
6. マイグレーションとユーザー所有データを確認する前に、上流機能のみのビルドへダウングレードしないでください。
