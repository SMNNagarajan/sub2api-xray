# Sub2API Xray v0.1.161-xray2 发行说明

发布日期：2026-07-20

上游基线：[Sub2API v0.1.161](https://github.com/Wei-Shaw/sub2api/releases/tag/v0.1.161)（`b8e844f4e`）

本版本将 Xray 分支更新到官方最新 `main`，同时保留用户私有资源工作台和代理运行时。本次没有新增 Xray 专属版本序列，因此 Xray 修订号继续使用 `2`。

## 变更内容

- 合并官方 v0.1.161 线路，包括网关、账号、图片、运维、鉴权缓存和入口拒绝处理等更新。
- 保留用户分组、账号、代理、代理订阅源、分配订阅和兑换码的所有者隔离。
- 保留标准代理与 Xray 代理、协议导入、公开代理认证信息脱敏、连接测试和质量检测。
- 在官方 `183`、`184` 迁移之后使用新的迁移编号，继续维护用户公开代理约束。
- 保持 `Sub2API Xray` 品牌和 `v<上游版本>-xray<修订号>` 版本格式，与官方更新检查渠道区分。

## 兼容性与安全

- 历史资源继续作为系统资源，不会在升级时自动分配给用户。
- 普通用户继续使用 `/my/*` 用户资源接口，不能通过管理员资源接口读取其他用户数据。
- 公开代理返回结果不会包含密码、节点密钥或其他私有认证字段。
- 订阅、账单和 API Key 鉴权缓存继续保留两条分支合并后的失效机制。
- 用户资源工作台仍由 `enable_user_resources` 控制，升级后的实例默认关闭。

## 升级说明

1. 备份 PostgreSQL、Redis、配置文件和代理订阅密钥。
2. 部署本版本镜像或二进制，并等待内置迁移执行完成。
3. 启用用户资源前，核对管理员权限和资源所有权。
4. 先分别用管理员账号和普通用户账号完成基本操作测试，再恢复正常流量。

在检查 Xray 专属迁移和用户私有资源数据前，不要降级到仅包含原版功能的构建。

## 验证结果

- 后端：Go `1.26.5` 下执行 `go test ./... -count=1` 通过。
- 前端：`vue-tsc --noEmit`、`pnpm test:run`（`188` 个文件、`1272` 个测试）和 `pnpm build` 通过。
- 最终 Docker 镜像和健康检查接口已在审核服务器部署前验证。

原版项目位于 [Wei-Shaw/sub2api](https://github.com/Wei-Shaw/sub2api)。本仓库是独立维护的 Xray 分支。
