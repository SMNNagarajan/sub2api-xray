# Sub2API Xray v0.1.161-xray2 修复及更新

发布日期：2026-07-22

## 官方版本同步

- 合并官方 v0.1.161 线路，保留该版本的网关、账号、图片、运维、鉴权缓存和入口安全更新。
- Xray 专属数据库变更使用迁移 `185`，避免与官方迁移 `183`、`184` 冲突。

## 代理订阅与协议兼容

- 订阅导入支持普通节点链接、Base64 订阅、Clash/Mihomo YAML，以及包含 `outbounds` 或 `endpoints` 的 sing-box JSON。
- 标准代理支持 HTTP、HTTPS、SOCKS、SOCKS5 和 SOCKS5H。
- Xray 类代理支持 VMess、VLESS、Trojan 和 Shadowsocks；Xray Core 负责前三类协议，并兼容 TCP、WebSocket、gRPC、HTTPUpgrade、XHTTP、TLS 与 REALITY 等常用组合。
- 新增受控 sing-box 运行时，承载 Shadowsocks，并支持 Hysteria、Hysteria2/Hy2、TUIC、AnyTLS、Naive（HTTPS/QUIC）和 WireGuard/WG。
- 兼容常见别名与分享链接格式；单个节点解析失败时返回对应错误，不会阻断其他有效节点导入。
- Docker 镜像同时内置 Xray Core 与 sing-box，并通过本地 SOCKS 入站统一提供给账号转发、连接测试和质量检测。

## 我的代理

- 添加标准代理、批量代理、节点链接、完整节点配置和机场订阅五类入口，交互和排版对齐管理员 IP 管理。
- 支持创建和维护代理订阅源、设置刷新间隔、手动同步，并同步新增、更新和停用订阅节点。
- 支持将自己的代理公开给所有用户使用，也可以让代理订阅源同步生成的节点自动公开；其他用户看不到订阅 URL，只能选择和使用公开节点。
- 非所有者查看公开代理时隐藏主机、端口、用户名、密码、节点原文、备用代理和其他敏感字段。
- 支持自有代理或公开代理作为下级/备用代理，也支持失败后直连；数据库同时阻止跨所有者私有代理引用和自引用。
- 增加单个连接测试、单个质量检测、批量连接测试和批量质量检测，并显示执行进度与检测结果。
- Xray 与 sing-box 节点检测使用隔离的临时运行实例，检测结束后自动清理。
- 导入或同步节点后自动排队执行一次质量检测；使用有界队列和固定工作线程，避免批量导入拖垮服务。
- 优化搜索栏、筛选、列设置、位置/质量/账号数展示、操作菜单和宽表格布局。

## 我的账号与管理员账号管理

- “我的账号”复用管理员账号管理的创建和编辑弹窗，并以用户作用域加载分组、代理、平台和账号类型。
- 完善 OAuth、API Key、Setup Token、Cookie、Service Account、Codex Session/PAT 等原版账号录入与导入流程。
- 增加刷新令牌、清除错误、查看使用记录、调度开关、批量刷新、批量清错、批量编辑、导入和导出等操作。
- 账号列表的筛选、列设置、容量、状态、用量窗口、代理、优先级、倍率和时间字段与管理员页面保持一致。
- 修复管理员创建、编辑或批量编辑账号时代理选择错误：系统账号只能使用系统代理，用户账号只能使用自己的代理或公开代理。
- “测试连接”复用管理员完整测试弹窗，支持模型选择、OpenAI 测试模式、SSE 实时输出、重试和图片预览。
- 测试提示词可以编辑，默认值为 `hi`；适配 OpenAI、Claude、Gemini、Grok、Bedrock、Vertex 和 Antigravity，Compact 模式自动隐藏提示词。
- 修复账号测试服务和令牌刷新服务未注入导致用户测试接口不可用的问题。

## 本轮补充修复与安全加固

### 用户资源生命周期与事务一致性

- 管理员删除或停用用户时，以同一数据库事务停用其私有分组、账号、代理、代理订阅源和兑换码，并撤销由该用户管理的订阅；内容审核自动封禁复用同一生命周期流程。
- 用户主动退订会记录撤销人，资源所有者不能重新激活用户已主动退出的订阅；管理员仍保留显式强制恢复能力。
- 订阅授权变更通过数据库 outbox 同步失效 API Key 鉴权缓存，避免 Redis 或跨实例通知短暂异常后继续使用旧权限。
- 分组倍率与 RPM 覆盖、账号字段与分组关系、代理订阅同步、缺失节点停用和调度通知均改为原子写入，任一步失败都会整体回滚。
- 用户资源删除、清错、刷新、批量修改和代理订阅同步会在事务内写入调度 outbox，避免数据库状态与调度缓存不一致。

### 所有权、隐私与网络安全

- 管理员创建、编辑和批量编辑账号时，账号字段、分组关系与代理归属在提交前统一校验，并在同一事务中保存，阻止跨用户分组或私有代理引用。
- 公开代理被其他账号或备用代理引用时禁止取消公开或删除；备用代理引用也纳入依赖计数和缓存失效范围。
- 公开代理响应继续隐藏节点原文与认证信息，并补充现代分享协议中的敏感字段清理；代理客户端缓存键仅保存认证信息摘要。
- 用户私有账号在创建、模型预览和真实请求阶段均强制公网目标策略，拒绝环回、私网、链路本地和云元数据地址；系统账号保持官方自定义上游行为。
- 正式部署恢复官方 URL 白名单默认值，自定义公网 API 渠道不再被静态主机名单误拦截；用户私有资源的独立 SSRF 防护保持启用。
- 含凭据的用户资源接口不记录请求体，API Key 鉴权同时拒绝访问不属于当前用户的私有标准分组。

### 账号能力与网关兼容

- “我的账号”增加上游模型同步、创建前模型预览、Gemini OAuth 能力查询和 Antigravity 默认模型映射，所有接口继续使用用户作用域权限。
- 创建与编辑弹窗按管理员/用户作用域加载模型、代理和 OAuth 能力，并完善 Codex 导入结果、Gemini OAuth 选项和模型白名单交互。
- 管理员账号管理与“我的账号”统一使用同一个连接测试弹窗；普通文本测试提示词可编辑，默认 `hi`，Compact 模式仍自动隐藏提示词。
- 增加 Grok `/messages/count_tokens` 兼容入口、Anthropic 请求 token 估算、免费账号滚动 24 小时额度识别和计费探测重试。
- 改进 Grok 多轮缓存身份、Claude Code session 识别、加密 reasoning 失败后的单次降级重试，以及 Anthropic thinking signature 与 Responses reasoning 的双向转换。
- 内容审核封禁、用户状态和私有资源停用保持原子一致；生命周期事务失败时不会只改变用户状态而遗留可用资源。

### 构建与部署

- 新增迁移 `186_user_resource_lifecycle_hardening.sql`，补充订阅撤销归因、代理备用引用约束和订阅鉴权缓存 outbox。
- Docker 构建上下文忽略本地 `.pnpm-store`，避免依赖缓存被重复打包或上传导致构建异常缓慢。

## 升级说明

1. 升级前备份 PostgreSQL、Redis、配置文件和代理订阅密钥。
2. 本次包含数据库迁移 `185_user_public_proxies.sql` 与 `186_user_resource_lifecycle_hardening.sql`，会增加代理订阅源公开标记、账号/备用代理所有权约束、订阅撤销归因和鉴权缓存失效 outbox。
3. Docker 镜像已经包含 Xray Core `26.3.27` 和 sing-box `1.13.14`；使用独立二进制部署时需要正确配置 `XRAY_BIN` 与 `SING_BOX_BIN`。
4. 部署后先核对管理员账号切换代理、普通用户代理导入、公开代理选择和账号连接测试。
5. 用户资源工作台仍由 `enable_user_resources` 控制，升级后的实例默认关闭。
6. 在确认迁移和用户私有资源数据前，不要降级到仅包含原版功能的构建。
