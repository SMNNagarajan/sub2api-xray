# Sub2API Xray v0.1.157-xray1 发行说明

发布日期：2026-07-16

上游基线：[Sub2API v0.1.157](https://github.com/Wei-Shaw/sub2api/releases/tag/v0.1.157)

这是 Sub2API Xray 独立版本与更新通道的首个发行版。在原版 `v0.1.157` 的基础上，加入用户私有资源管理、订阅分发和内置 Xray 代理运行时。

## 主要内容

- 新增完整的用户私有资源工作台，普通用户不需要也不能调用管理员接口。
- 新增标准代理与 Xray 代理池、节点导入和机场订阅同步。
- 新增用户自行分配订阅与兑换码分发能力。
- 新增号池健康、退订、账号使用记录和上游错误详情。
- 新页面与管理员界面体验对齐，并改善弹窗、表格和小窗口适配。

## 新增功能

### 用户私有资源

- 我的分组：创建和管理私有分组，支持路由、额度、倍率、RPM、模型、fallback 和平台配置。
- 我的账号：维护私有账号池，支持各平台接入方式、测试、刷新、导入导出和批量操作。
- 我的代理：维护私有代理，也可以选择管理员公开的代理，但不会获得其认证信息。
- 分配订阅：使用自己的分组分配、续期、重置、撤销和恢复订阅。
- 我的兑换码：分发订阅兑换码，支持重复兑换和查看兑换详情。

### Xray 代理运行时

- 支持 `vmess`、`vless`、`trojan`、Shadowsocks、SOCKS 和 HTTP 节点链接。
- 支持 Base64 节点订阅和常见 Clash YAML 订阅。
- 为 Xray 节点创建受控的本地 SOCKS 入站，账号流量和代理测试共用同一转发链路。
- 批量导入时逐条保存有效节点并返回行级错误，不因单个错误丢弃整批数据。

### 订阅与诊断

- 在我的订阅中显示可用、限流和总账号数量。
- 订阅用户可以退订自己的订阅。
- 使用记录增加“我的使用”和“我的账号”两种视角。
- 上游错误按当前用户拥有的资源过滤，并对敏感信息进行脱敏。

## 安全与隐私

- 私有分组、账号、代理、兑换码、代理订阅源和管理中的订阅均在服务层与仓储层执行所有者隔离。
- 普通用户不能把账号、路由或 fallback 分组指向其他用户的私有资源。
- 公开代理的密码、认证信息和节点密钥不会返回给普通用户。
- 分配、撤销、恢复、兑换和退订后会刷新订阅与账单缓存。
- 禁用用户时同步停用其私有资源和分发路径，不会把资源转换成系统资源。
- 用户资源工作台由 `enable_user_resources` 控制，升级后的实例默认关闭。

## 品牌与更新

- 产品名称改为 `Sub2API Xray`。
- 版本格式改为 `v<上游版本>-xray<修订号>`。
- 更新检查、回滚说明、发行链接和容器镜像均改用 `SMNNagarajan/sub2api-xray`。
- 容器镜像地址为 `ghcr.io/smnnagarajan/sub2api-xray`。

## 升级说明

1. 升级前备份 PostgreSQL、Redis、配置文件和代理订阅密钥。
2. 正常执行数据库迁移。历史资源继续作为系统资源，不会自动分配给任何用户。
3. 配置 Xray 运行时，或使用已经内置 xray-core 的本项目 Docker 镜像。
4. 启用 `enable_user_resources` 前，先核对管理员权限和现有资源所有权。
5. 启用后先用普通用户创建私有代理或分组并完成调用测试，再逐步开放使用。

在未检查 Xray 专属迁移和用户私有资源数据前，不要直接降级到仅包含原版功能的构建。
