# Sub2API Xray v0.1.157-xray1

Release date: 2026-07-16

Base release: [Sub2API v0.1.157](https://github.com/Wei-Shaw/sub2api/releases/tag/v0.1.157)

This is the first release under the Sub2API Xray version and update channel. It combines upstream `v0.1.157` with private user resource management, subscription distribution, and an embedded Xray proxy runtime.

## Highlights

- Added a complete user-owned resource workspace without exposing administrator APIs to ordinary users.
- Added standard and Xray-backed proxy pools, node import, and proxy-provider subscription synchronization.
- Added user-managed subscription assignment and redeem-code distribution.
- Added account-pool health, unsubscribe support, account usage records, and upstream error details.
- Aligned the new user pages with the administrator UI and improved responsive modal and table behavior.

## Added

### User-owned resources

- My Groups: create and manage private groups with routing, quota, multiplier, RPM, model, fallback, and platform settings.
- My Accounts: maintain private account pools with platform-specific setup, testing, refresh, import/export, and batch operations.
- My Proxies: maintain private proxies and select administrator-published public proxies without receiving their credentials.
- Assigned Subscriptions: assign, renew, reset, revoke, and restore subscriptions backed by user-owned groups.
- My Redeem Codes: distribute subscription redeem codes, including repeatable codes and redemption details.

### Xray proxy runtime

- Supports `vmess`, `vless`, `trojan`, Shadowsocks, SOCKS, and HTTP node links.
- Supports Base64 subscription lists and common Clash YAML subscriptions.
- Creates managed local SOCKS inbounds for Xray nodes and reuses the same path for account traffic and proxy tests.
- Imports valid nodes independently and reports line-level errors without discarding the full batch.

### Subscription and diagnostics

- Shows available, rate-limited, and total account-pool counts for user subscriptions.
- Allows a subscriber to unsubscribe from their own subscription.
- Adds user and user-owned-account usage perspectives.
- Adds redacted upstream error details scoped to the current user's resources.

## Security and Privacy

- Private groups, accounts, proxies, redeem codes, proxy sources, and managed subscriptions are owner-scoped in both service and repository layers.
- Ordinary users cannot bind accounts, routes, or fallback groups to another user's private resources.
- Public proxy credentials and node secrets are not returned to ordinary users.
- Subscription and billing caches are invalidated after assignment, revocation, restoration, redemption, or unsubscribe operations.
- Disabling a user also disables their private resources and managed distribution paths instead of converting them to system resources.
- The user resource workspace is gated by `enable_user_resources`, which defaults to disabled for upgraded installations.

## Branding and Updates

- Product name changed to `Sub2API Xray`.
- Version format changed to `v<upstream>-xray<revision>`.
- Update checks, rollback instructions, release links, and container images now use `SMNNagarajan/sub2api-xray`.
- The container image is `ghcr.io/smnnagarajan/sub2api-xray`.

## Upgrade Notes

1. Back up PostgreSQL data, Redis data, configuration, and proxy subscription secrets before upgrading.
2. Apply the normal database migrations. Existing resources remain system-owned; no historical resource is assigned to a user automatically.
3. Configure the Xray runtime or use the project Docker image, which includes xray-core.
4. Verify administrator access and resource ownership before enabling `enable_user_resources`.
5. Enable the feature, create a private proxy or group, and test with a non-administrator account before wider use.

Do not downgrade to an upstream-only build without first reviewing the Xray-specific migrations and user-owned resource data.
